# 要件定義

対応する Rules は `docs/rules/requirements.md` を参照する。

## 1. 機能要件

### 1.1 入力

- アプリは画像ファイルを 1 枚だけ受け付ける
- 対応する読み込み方法は次の 3 つとする
- ファイル選択ダイアログ
- ドラッグアンドドロップ
- 起動時のコマンドライン引数

### 1.2 表示

- 読み込んだ画像を UI に表示する
- 現在の入力状態をユーザーがすぐ分かるようにする
- 処理中は状態表示を行う
- 処理完了後は出力結果を確認できるようにする

### 1.3 変換

- AI を使って 2 倍、3 倍、4 倍拡大を行う
- 倍率は UI から選択できるようにする
- GPU を使って推論する
- 処理中に同じジョブを二重実行できないようにする

### 1.4 保存

- 出力画像を任意の保存先へ保存できる
- 初期の保存形式は PNG と JPEG を対象にする
- 既定の保存形式は PNG とする

## 2. 非機能要件

### 2.1 操作性

- デモっぽい画面ではなく、常用できるデスクトップアプリの見た目にする
- メイン操作は数ステップで完了するようにする
- 不正な操作は UI 側で抑止する

### 2.2 応答性

- 推論中に UI スレッドを止めない
- 処理開始、処理中、処理完了、失敗の状態が分かるようにする

### 2.3 拡張性

- 後で複数画像処理へ拡張できる構造にする
- 今は不要な抽象化は増やしすぎない
- ただし UI と推論ロジックは分離する

### 2.4 保守性

- 画像読み込み経路を 1 本化する
- 推論実行経路を 1 本化する
- モデルの差し替えが局所的に済むようにする

## 3. 入出力要件

### 3.1 初期対応入力形式

- PNG
- JPEG / JPG
- WebP

### 3.2 初期対応出力形式

- PNG
- JPEG / JPG

## 4. エラー時の扱い

- 非対応形式を読み込んだ場合は短く分かりやすいエラーを表示する
- GPU 推論に失敗した場合は処理失敗を明示する
- 保存に失敗した場合は原因を推測しやすい文言にする

## 5. 現時点で範囲外とする項目

- 複数画像キュー
- モデル切り替え UI
- タイル分割 UI
- 詳細画質パラメータ調整
- バックグラウンドの自動監視
- 履歴管理

## 6. 受け入れ条件

以下を満たしたら MVP の要件を満たしたと判断する。

1. 画像を 1 枚選択してプレビューできる
2. ドラッグアンドドロップでも同じ読み込み経路で受け付けられる
3. 起動引数で画像パスを渡しても同じ読み込み経路で開ける
4. 2 倍、3 倍、4 倍の AI 拡大を GPU で実行できる
5. 処理中に UI が固まらない
6. 出力画像を保存できる
