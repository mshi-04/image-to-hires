# 要件定義

対応する Rules は `docs/rules/requirements.md` を参照する。

## 1. 機能要件

### 1.1 入力

- アプリは画像ファイルを 1 枚以上受け付ける
- 対応する読み込み方法はファイル選択ダイアログのみとする
- ドラッグアンドドロップは受け付けない
- 複数ファイルが渡された場合は、受け付けた順にキューへ積んで順次処理する
- 空入力はエラーにせず無視する
- 起動時のコマンドライン引数は受け付けない

### 1.2 表示

- 読み込んだ画像のファイル名一覧を UI のテキストボックスへ表示する
- キューに積まれた入力一覧と現在処理中の対象を UI に表示する
- 現在処理中の対象と進行状況を UI の状態表示へ表示する
- 現在の入力状態をユーザーがすぐ分かるようにする
- 処理中は状態表示を行う
- 処理完了後は出力結果を確認できるようにする

### 1.3 変換

- AI 拡大 (2x / 3x / 4x) を行う
- デノイズは `なし / 0 / 1 / 2 / 3` から選択できるようにする
- 倍率は UI から選択できるようにする
- GPU 推論は 2x 以上で行う
- 処理中に同じジョブを二重実行できないようにする
- キューに積まれたジョブは 1 件ずつ順に実行する

### 1.4 保存

- 出力画像は入力画像と同じフォルダへ自動保存する
- 出力形式は入力画像の拡張子を維持する
- 初期の対応保存形式は PNG、JPEG / JPG、WebP とする
- 出力ファイル名は `{stem}-denoiseXx-upYx{suffix}` とする
- 保存時の既定品質は PNG が可逆、JPEG が `quality=100` かつ `subsampling=0`、WebP が `lossless=True` かつ `quality=100` とする
- デノイズ「なし」を選んだ場合、`X` は `-1` を使う
- デノイズ「なし」は 1x 廃止後も有効で、ファイル名の `X` は `-1` で表す
- JPEG 保存時に入力画像または出力画像が透過を持つ場合は、白背景で合成して保存する

## 2. 非機能要件

### 2.1 操作性

- デモっぽい画面ではなく、常用できるデスクトップアプリの見た目にする
- `qt-material` のライトテーマを活用し、清潔感のある Material 風のモダンな UI にする
- メイン操作は数ステップで完了するようにする
- 不正な操作は UI 側で抑止する
- キューの進行状況と現在処理中の対象が分かるようにする

### 2.2 応答性

- 推論中に UI スレッドを止めない
- 処理開始、処理中、処理完了、失敗の状態が分かるようにする
- GPU 都合で複数画像の並列生成は行わず、順次処理のまま総処理時間を短縮する

### 2.3 拡張性

- 複数画像キューを無理なく扱える構造にする
- 今は不要な抽象化は増やしすぎない
- ただし UI と推論ロジックは分離する

### 2.4 保守性

- 画像読み込み経路を 1 本化する
- 推論実行経路を 1 本化する
- 単一画像処理と複数画像キュー処理で、1 ジョブ単位の実行ロジックを共有する
- モデルの差し替えが局所的に済むようにする
- 高速化は Real-CUGAN 実行設定、一時ファイル再利用、保存オーバーヘッド削減を優先し、責務分離を崩さない

## 3. 入出力要件

### 3.1 初期対応入力形式

- PNG
- JPEG / JPG
- WebP

### 3.2 初期対応出力形式

- PNG
- JPEG / JPG
- WebP

## 4. エラー時の扱い

- 非対応形式を読み込んだ場合は短く分かりやすいエラーを表示する
- GPU 推論に失敗した場合は処理失敗を明示する
- 保存に失敗した場合は原因を推測しやすい文言にする
- キュー内の 1 件が失敗しても、その失敗を表示した上で残りのジョブ処理は継続できるようにする

## 5. 現時点で範囲外とする項目

- GPU を使った複数画像の並列生成
- モデル切り替え UI
- タイル分割 UI
- 詳細画質パラメータ調整
- バックグラウンドの自動監視
- 履歴管理

## 6. 受け入れ条件

以下を満たしたら MVP の要件を満たしたと判断する。

1. 画像を 1 枚以上選択して入力一覧を確認できる
2. 複数画像を選択した場合に、キューへ積まれて順次処理される
3. ファイル選択から読み込み経路へ受け付けられる
4. 2 倍、3 倍、4 倍の AI 拡大を GPU で実行できる
5. 処理中に UI が固まらない
6. 出力画像が入力画像と同じフォルダへ自動保存される
7. 出力画像が入力画像の拡張子を維持して保存される
8. 透過画像を JPEG で保存した場合に白背景で合成される
