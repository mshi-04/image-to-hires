# プロジェクト概要

## 1. 目的

このプロジェクトは、画像を AI で高品質に 2 倍、3 倍、4 倍へ拡大し、複数ファイルをキュー方式で順次処理できる Windows 向け GUI アプリを作るためのものです。

狙う体験は次のとおりです。

- エクスプローラーから画像を開ける
- 入力ファイル一覧やキュー状態を確認できる
- GPU を使って AI アップスケールを実行できる
- 出力画像が入力元と同じ場所へ自動保存される

初期段階ではインストーラーや配布自動化を目的にしません。ただし、ローカル起動は `.exe` を主導線とし、あとで配布対応をしやすいように構造と UI は最初から整理して実装します。

## 2. 現在の MVP 範囲

### 2.1 対象

- Windows デスクトップアプリ
- NVIDIA GeForce GPU 前提
- 画像を 1 枚または複数枚処理
- 複数ファイルはキュー方式で順次処理
- 2 倍、3 倍、4 倍拡大に対応
- GUI 操作前提

### 2.2 入力方法

- ファイル選択ダイアログ
- ドラッグアンドドロップは MVP 対象外

### 2.3 MVP に含めないもの

- GPU を使った複数画像の並列生成
- 動画対応
- 学習機能
- インストーラー作成
- 配布自動化
- CPU を主対象にした実装

## 3. 現時点の方針

- 言語は Python を第一候補とする
- GUI は PySide6 を使う
- 推論は PyTorch + CUDA を使う
- 構成は `ui / domain / infrastructure` を基本にする
- ユースケースは `domain/usecase` に置く
- 既存の学習済みモデルを使い、自前学習はしない
- 1 ジョブ単位の処理を積み上げて、複数ファイルを順次処理できる形にする
- GPU 都合で複数画像の並列生成は採用しない
- 高速化は順次処理の範囲で行い、Real-CUGAN の実行設定、一時ファイル再利用、保存オーバーヘッド削減を優先する
