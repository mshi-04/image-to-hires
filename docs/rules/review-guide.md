# レビューガイド Rules

## 1. 優先順位

- バグ、回帰、設計崩れを最優先に見る
- 仕様逸脱とアーキテクチャ違反を先に見る
- 好みだけの指摘は後回しにする

## 2. 指摘の書き方

- 事実、問題、修正案を分けて書く
- 指摘は `Critical / Suggestion / Nitpick` に分類する
- 項番を振って列挙する

## 3. 文書確認

- 仕様変更は `requirements`
- 設計変更は `architecture`
- 実装ルール変更は `coding-standards`
- テスト方針変更は `unit-test-guidelines`
- Git 運用変更は `git-guidelines`
