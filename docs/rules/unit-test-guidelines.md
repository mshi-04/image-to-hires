# UnitTest Rules

## 1. 対象

- 契約、分岐、状態遷移を守るテストを書く
- CPU で回る境界テストを優先する
- UI の見た目ではなくロジックを確認する

## 2. 方針

- 1 テスト 1 振る舞いにする
- テスト本文では `Arrange / Act / Assert` の AAA コメントを必須とする
- GPU 実機依存は最小化する
- 正常系だけでなく失敗系も追加する
- 重複ケースは `subTest` やヘルパーで削減し、保守性を優先する
- 廃止した仕様のテストケースは残さない

## 3. モック

- 境界でだけモックする
- 実ファイルや GPU は必要最小限にする
- モデル品質そのものは UnitTest の主対象にしない
