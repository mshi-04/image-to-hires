# Git 運用ガイド

対応する Rules は `docs/rules/git-guidelines.md` を参照する。

## 1. 目的

この文書は、このリポジトリで Git を使って変更を積み、PR を進め、レビューへ応答するときの運用ルールを定めるものです。

目的は次の通りです。

- 履歴と PR 上のやり取りを追いやすくする
- レビュー指摘への対応漏れを防ぐ
- CodeRabbit を含む自動レビューとの往復を明確にする

## 2. コミット運用

- コミットは 1 つの意図または修正単位で分ける
- コミットタイトルは日本語で簡潔に書く
- 補足が必要なら、コミット本文も日本語で理由と影響範囲を書く
- 無関係な変更を 1 つのコミットへ混ぜない

## 3. PR 運用

- PR タイトルは日本語で、何を変える PR かすぐ分かる形にする
- PR 本文と補足コメントも日本語で書く
- PR 本文には、少なくとも概要、確認方法、未確認事項または既知の制約を残す
- UnitTest や Lint を未実行にした場合は、その理由を日本語で明記する

## 4. レビューコメント対応

- レビュアーのコメントには、該当する会話スレッドへ直接返信する
- 返信には、対応したコミット ID を添える
- 修正済みなら、何をどう直したかを短く日本語で返す
- 修正しない場合も、理由や判断を日本語で返す
- コメントへの応答を残さずに push だけして終わらせない

## 5. CodeRabbit 対応

- CodeRabbit の指摘コメントには、該当スレッドへ直接返信する
- リプライ内容には、対応したコミット ID を添える
- CodeRabbit の指摘を採用して修正した場合は、該当スレッドまたは PR 上で対応内容を日本語で残す
- CodeRabbit の指摘修正後は、CodeRabbit に再レビューを依頼する
- 複数の指摘をまとめて直した場合でも、何を直したかをまとめて残してから再レビューを依頼する
- 指摘を採用しない場合は、その理由を日本語で残す

## 6. 文書更新ルール

Git 運用を変えた場合は、この文書と `AGENTS.md` の文書マップ、必要なら `docs/review-guide.md` を一緒に更新する。
