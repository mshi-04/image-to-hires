# 実装計画

## Rules

- 実装順は MVP 完成を最短で進める順にする
- 先送りする項目は範囲外か後続タスクかを明示する
- 計画を変えたら要件とアーキテクチャに反していないか確認する

## 1. 実装フェーズ

### フェーズ 1: 土台作成

- Python プロジェクトの最小構成を作る
- `src` 配下に責務別のディレクトリを作る
- PySide6 で空のメインウィンドウを表示する
- 画像読み込みの共通入口を用意する

### フェーズ 2: 入力まわり

- ファイルダイアログから画像を読み込む
- ドラッグアンドドロップで画像を読み込む
- 起動引数で受け取った画像パスを読み込む
- プレビューを表示する

### フェーズ 3: 推論 MVP

- モデル読み込み処理を追加する
- GPU 推論の実行経路を追加する
- 2 倍、3 倍、4 倍変換をバックグラウンドで動かす
- 結果プレビューを表示する

### フェーズ 4: 仕上げ

- 保存ダイアログを追加する
- エラー表示を整える
- 実行中のボタン制御を整える
- 最低限の設定保持を入れる

## 2. 実装上の優先順位

優先順位は次の通り。

1. 画像を安全に読み込める
   完了条件: ファイル選択、ドラッグアンドドロップ、起動引数の 3 経路が同じ読み込み入口を通り、非対応入力を弾ける。
2. UI が固まらずに推論できる
   完了条件: 推論中も UI スレッドが応答し、開始中と完了後の状態表示が確認できる。
3. 出力を保存できる
   完了条件: PNG と JPEG の保存導線が動作し、保存失敗時のエラー表示も確認できる。
4. 見た目と導線を整える
   完了条件: 主要操作が数ステップで辿れ、入力状態と処理結果が画面上で分かる。
5. 将来拡張しやすい形へ整理する
   完了条件: `ui / domain / infrastructure` の責務分離を保ち、単一画像処理の入口が `domain/usecase` に集約されている。

## 3. 実装時の判断基準

- 早く動くものを作るが、責務分離は崩さない
- 今不要な汎用化はしない
- ただし後で詰む近道は避ける
- UI 操作と推論ロジックを混ぜない

## 4. 将来タスク

MVP 完了後の候補は次の通り。

- 複数画像の一括処理
- タイル分割による大画像対応
- モデル切り替え
- 画質モード追加
