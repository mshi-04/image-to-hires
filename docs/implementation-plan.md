# 実装計画

## 1. 実装フェーズ

### フェーズ 1: 土台作成

- Python プロジェクトの最小構成を作る
- `src` 配下に責務別のディレクトリを作る
- PySide6 で空のメインウィンドウを表示する
- 画像読み込みの共通入口を用意する

### フェーズ 2: 入力まわり

- ファイルダイアログから画像を読み込む
- 複数ファイルをキューへ積む
- ファイル名一覧とキュー状態を表示する

### フェーズ 3: 推論 MVP

- モデル読み込み処理を追加する
- GPU 推論の実行経路を追加する
- 2 倍、3 倍、4 倍変換をバックグラウンドで順次動かす
- 処理結果を状態表示で確認できるようにする

### フェーズ 4: 仕上げ

- 自動保存の導線を整える
- エラー表示を整える
- 実行中のボタン制御を整える
- 最低限の設定保持を入れる

## 2. 実装上の優先順位

優先順位は次の通り。

1. 画像を安全に読み込める
   完了条件: ファイル選択の読み込み入口から複数ファイルをキューへ積めて、非対応入力を弾ける。
2. UI が固まらずに推論できる
   完了条件: 推論中も UI スレッドが応答し、キューの進行状態と開始中、完了後の状態表示が確認できる。
3. 出力を保存できる
   完了条件: 入力画像と同じディレクトリへの自動保存が動作し、PNG、JPEG、WebP の出力が扱え、透過画像の JPEG 保存で白背景合成が行われ、保存失敗時のエラー表示も確認できる。
4. 見た目と導線を整える
   完了条件: 主要操作が数ステップで辿れ、入力状態、キュー状態、処理結果が画面上で分かる。
5. 将来拡張しやすい形へ整理する
   完了条件: `ui / domain / infrastructure` の責務分離を保ち、1 ジョブ単位の処理と複数ファイルキュー管理が `domain/usecase` に集約されている。

## 3. 実装時の判断基準

- 早く動くものを作るが、責務分離は崩さない
- 今不要な汎用化はしない
- ただし後で詰む近道は避ける
- UI 操作と推論ロジックを混ぜない

## 4. 将来タスク

MVP 完了後の候補は次の通り。

- 順次処理の性能改善（Real-CUGAN 実行設定、一時ファイル再利用、保存オーバーヘッド削減）
- タイル分割による大画像対応
- モデル切り替え
- 画質モード追加

## 5. 文書更新ルール

実装フェーズ、優先順位、完了条件を変えた場合は、この文書と `AGENTS.md`、必要な関連文書を一緒に更新する。
