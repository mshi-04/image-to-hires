# Material風 UI実装プラン (PySide6)

## 1. 目的と方針

PySide6 で Windows 向けデスクトップアプリとして自然に動作しつつ、「Material風」の美しくモダンですっきりした UI を、**サードパーティライブラリを活用して「無理なく」「最少のコード量」で実装**する計画です。
**なお、中途半端な改修による複雑化を避けるため、従来のUIコードは完全に破棄し、全面刷新（ゼロからの再構築）を行います。**
※ダークテーマは採用せず、**ライトテーマ専用** とします。

## 2. アプローチの選定

PySide6 向けに Material Design テーマを一括適用できるライブラリ **`qt-material`** を採用します。

- **メリット**: `pip install qt-material` で導入でき、わずか2行のコードで手軽にアプリ全体を Material 風の美しい UI に一変させることができます。
- **メリット**: ボタン、ドロップダウン、プログレスバーなどの見栄えが自動でフラット＆モダンになるため、独自のQSS微調整にかかる保守の手間が省けます。
- **メリット**: 複数のライトテーマ（Light Blue, Light Amber など）が用意されているため、明るく清潔感のあるデザインを容易に実現できます。現在はユーザーの要望に基づき「白背景と青いボタン」を実現する `light_blue.xml` を採用します。

## 3. `qt-material` の適用イメージ（ライトテーマ）

| UI 要件 | PySide6 ウィジェット | `qt-material` (ライトテーマ) 適用後の見た目 |
| :--- | :--- | :--- |
| **全体背景** | `QMainWindow` | 清潔感のある白・ライトグレー系の背景色。 |
| **ボタン類** | `QPushButton` | フラットなベタ塗り（白文字）またはアウトライン。ホバー時に色合いが変化。 |
| **プルダウン** | `QComboBox` | 角丸でフラットなドロップダウン。 |
| **キューリスト** | `QListWidget` | 選択状態のハイライト色がプライマリカラーと自動連動し、すっきりとしたリストになる。 |
| **進行状況** | `QProgressBar` | フラットでモダンなインジケーター。 |

## 5. 実装フェーズ

### フェーズ 0: 実装用ブランチの作成
- `develop` ブランチから本件のUI実装用（Materialテーマ適用およびドキュメント更新など）の新しい feature ブランチを作成する。

### フェーズ 1: パッケージ導入とライトテーマ適用
- `pip install qt-material` を実行。
- `main.py` の GUI 初期化位置で、ライトテーマとして `light_blue.xml` を読み込む（白背景・青ボタン構成）。

### フェーズ 2: UI モックのレイアウト構築
- **既存のUIコード（`src/ui/` 配下）は一旦すべて破棄し、新しい実装に置き換える。**
- 既存のアーキテクチャ `src/ui/` に合わせて新規にウィジェットを配置する。
- 「リストアイテムの余白」などのレイアウト微調整以外は、`qt-material` の標準の見た目をそのまま使用。

### フェーズ 3: ロジックとの結合
- ユースケース層・インフラ層の処理とつなぎ、プログレスバーの更新や状態表示の切り替えがスムーズに行えることを確認。

### フェーズ 4: AI用文書（各種仕様書）の更新
- UI 方針、外部ライブラリ（`qt-material`）の導入、および「ライトテーマ固定」という決定事項に伴い、以下のAI・開発者向け文書を更新する。
  - `docs/technical-design.md` (または `docs/rules/` 等): GUI実装方針・ライブラリ依存情報を追記する。
  - `docs/requirements.md`: UIにおける制約（ライトテーマ固定など）の要件を追加する。
